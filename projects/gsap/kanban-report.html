Introduction

Kanban Report is an Enterprise Single Web Application that uses Azure DevOps REST API to quickly generate report.

The problem was supervisors on our project are generating routinary report via Excel that takes 15-30 minutes per day. Data has to be placed on the correct cell ranges or update it to avoid errors on formulas. Items having challenges are not automatically highlighted for the supervisors to easily identify them. Overview has to be manually created for each seven areas to monitor.

Goal & Approach

The goal was to generate routinary report in just a few minutes. Data is extracted real-time with the use of Azure DevOps REST API. Web application algorithms will generate an Overall table and easily identify items having challenges.

The approach I did was to conduct research on what is the best type of Azure DevOps API to use based on the requirements. I selected Work Items API type because it is able to get all the details and converts those to JSON data. I then used an AJAX request to get the JSON data and post it dynamically into a table. I've added JavaScript and JQuery algorithms to generate an Overview and highlight items that exceeded based on the requirements. Lastly for better look, I implemented Glassmorphism which is a UI Design Trend in 2021.

Full list of technologies used: HTML, CSS, JavaScript, AJAX, REST API, JSON and JQuery

Design Decision: User-Centered and Glassmorphism

Conclusion

Generating routinary report just took 1-2 minutes. Items having challenges are easily identified thru red highlights. Time saved is used on other important tasks.
