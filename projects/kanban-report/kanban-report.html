Generate routinary reports and identify items having challenges in just a few minutes!

Introduction

Kanban Report is an Enterprise Single Web Application that uses Azure DevOps REST API to quickly generate report.

The problem was supervisors on our project are generating routinary report via Excel that takes 10-20 minutes per day. Data has to be placed on the correct cell ranges or update it to avoid errors on formulas. Items having challenges are not automatically highlighted for the supervisors to easily identify them. Overview has to be manually created for each seven areas to monitor.



Goal & Approach

The goal was to generate routinary report in just a few minutes. Data is extracted real-time with the use of Azure DevOps REST API. Web application algorithms will generate an Overall table and easily identify items having challenges.

On the design, I first conducted survey to identify the common pain points of the supervisors. I then performed interviews so I can gain a qualitative insights and list down the requirements. After the requirements gathering, I've designed the web app to display curated and relevant information per table. I continuously conduct Usability Testing to ensure that users can navigate the web app easily.

On the development, I conducted research on what is the best type of Azure DevOps API to use based on the requirements. I selected Work Items API type because it is able to get all the details and converts those to JSON data. I then used an AJAX request to get the JSON data and post it dynamically into a table. I've added JavaScript and JQuery algorithms to generate an Overview and highlight items that exceeded based on the requirements. Lastly for better look, I implemented Glassmorphism which is a UI Design Trend in 2021.

Full list of technologies used: HTML, CSS, JavaScript, AJAX, REST API, JSON and JQuery

UX Methods: Surveys, Interviews, and Usability Testing

Conclusion

Generating routinary report just took 1-2 minutes. Items having challenges are easily identified thru red highlights. After a week of releasing the MVP, managament and supervisors has supported on using this for reporting. Time saved is used on other important tasks.
